PoliteDB is a **modern, native database client** focused on **fast navigation**, **safe schema changes**, and a **pleasant keyboard-first workflow**.

It is designed to feel at home on macOS, with secure credential storage via the system **Keychain**, an offline‑friendly architecture, and a clean UI that stays out of your way while you work.

<Callout>
  This documentation explains what you can do with PoliteDB, how the main concepts fit together, and
  where to look next as you start using it day to day.
</Callout>

## What you can do with PoliteDB

- **Manage connections**: Create, edit, and organize connection profiles with tags, SSH, and SSL settings.
- **Work in tabs**: Open one or more connections into tabbed workspaces with their own layout and history.
- **Explore schemas**: Browse databases, schemas, tables, and functions from a structured navigator.
- **Run SQL quickly and safely**: Edit and execute SQL with autocomplete, history, and explain support.
- **Inspect data**: View table data with filtering, sorting, pagination, and quick export.
- **Keep credentials safe**: Store passwords and keys in the macOS Keychain instead of plain text.

## Supported databases

PoliteDB is built for teams that work with more than one engine. From a single interface you can connect to:

- PostgreSQL
- MySQL
- Redis

The exact feature set can vary slightly per engine, but the overall workflow—profiles, workspaces, editors, and history—remains the same.

## How the app is organized

At a high level, the app is split into:

- A **home screen** for managing connection profiles and keychain items.
- One or more **connection workspaces**, each opened as a tab for a specific profile.
- A **smart SQL editor** with history, multiple tabs, and inline results.

You do not need to understand the internals to be productive, but if you’re curious about the architecture and stores that power these screens, see the [Architecture](/essentials/architecture) and [Workspace](/essentials/workspace) pages.
