PoliteDB is designed to be comfortable for keyboard‑driven users.

Keyboard shortcuts help you move around the app, run queries, and manage windows without relying on the mouse.

## Working in the SQL editor

In the SQL editor you can typically:

- Run the current query or selection with a single shortcut.
- Open a new SQL window tied to the current connection.
- Switch focus between the editor and results using the keyboard.

These shortcuts let you iterate on queries rapidly while keeping your hands on the keyboard.

## Navigating the workspace

Across a connection workspace you can use shortcuts to:

- Jump between open windows (for example, different SQL editors and table views).
- Toggle side and bottom panels, such as the schema navigator or query history.
- Move between connection tabs at the top of the app.

This makes it easy to manage complex sessions with multiple connections and windows.

## General guidance

To get the most from keyboard shortcuts:

- Learn the core actions you use every day, such as run query, open new window, and switch window.
- Use shortcuts for navigation and layout changes, relying on the mouse mainly for precise datagrid work.
- Keep an eye on tooltips and menus, which often display the associated shortcut for a given action.

Over time, this can make PoliteDB feel more like a text editor or IDE than a traditional point‑and‑click database client.
