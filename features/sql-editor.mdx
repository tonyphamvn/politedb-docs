The **SQL editor** is the main place where you write and run queries in PoliteDB.

It is designed for a fast, keyboard‑driven workflow that keeps your queries and results close together.

## Editor layout

When you open a SQL window in a workspace, you see:

- A clean text editor focused on SQL.
- Controls to run the current query or selection.
- A results area that appears alongside or beneath the editor, depending on your layout.

This lets you quickly adjust a query and see the effect without leaving the current context.

## Multiple editor windows

Within a single connection tab you can open multiple SQL editor windows:

- Use them to separate different tasks (for example, exploration, migrations, and diagnostics).
- Keep reference queries open in one window while you experiment in another.
- Close windows that you are finished with to keep the workspace tidy.

Each editor window is scoped to the active connection, so you do not have to reconfigure connections when switching between them.

## Typical workflows

Common ways to use the SQL editor include:

- Ad‑hoc exploration with `SELECT` statements and quick filters.
- Maintenance scripts for analyzing or cleaning data.
- Schema changes and migration‑style statements.

You can combine the editor with query history, the datagrid, and backups to build safe, repeatable workflows around your database.
